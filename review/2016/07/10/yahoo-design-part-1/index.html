Review of "Yahoo Design Pattern Library" Part I

The library is pretty old. May be it is a good document for us to review the design of commonly used UI/X design pattern. This library explains patterns clearly. It talks about “What it solve”, “When to use”, “How it solve”, “Why we use it”.

Layout

Grid System

Grid system standardize the size of elements

  • What it solve
    • The consistency among common page elements, page width, division of space, ad usage and code base.
  • When to use
    • Creating/managing a large quantity of web pages
    • Web pages are created by different groups and individuals
  • Why use this
    • Templages reduces designers’ and engineers’ effort
    • Consistency contributes to a cohesive brand and user expereience
  • Accessibility
    • Consistency make the position of page elements easy to predict

Navigation

Accordion

An accordion is a group of collapsible panels with links and selectable items inside.

  • What it solve
    • Give user access to a large quantity of item without overwhelming them
  • When to use
    • The space is constrained
    • The number of options is large
    • The list of items can be logically grouped into smaller, roughly equal sized chunks
  • How to solve
    • Prsent a two-tiered set of options: the top level is category, the second level is options
    • Only expand one default panel to help user focus
  • Tips
    • Expand the main category
    • Avoid nested accordion
    • Expand all categories when javascript is disabled
    • Use height:0 instead of display:none to hide the folded accordion
      • For screen reader

Alphanumeric Filter Links

Grouped links alphabetically. Show groups in different pages.

  • What it solve
    • Help user to find resources alphabetically within a large data set
  • When to use
    • When user know the first letter of result
  • Note
    • The idea is to reduce the length of page
    • I think this solution is outdated

Breadcrumbs

Displaying a horizontal list of labels that reflect the location of current page in the hierarchy of a website.

  • What it solve
    • Help users navigate up
    • Help users to understand where they is in relation to the rest of the site.
  • When to use
    • The website have lots of layer
    • Users are difficult the navigate between different layer
    • Users get into the webpage from external resource
      • Such as search engine
      • User need to get context of current page
  • How to solve
    • Showing a horizontal tag. The tags are the page levels.
  • Tips
    • Don’t use “Home” for topmost page, use specific name
      • The user may not know what is the website about, the specific name give extra information
    • The breadcrumb label should mach the corresponding page title

Pagination

  • There are two kinds of pagination
    • Item pagination
    • Search pagination
  • Disable the hyperlink of current page in pagination bar for accessibility

Tabs

For navigating stacked panes

Module Tabs

Show the content in a small “window” with multiple tabs. Update the content (switch to different pane) by javascript.

Note:

  • Clicking the tab should not affect other component
  • The categories title is short and informative
  • It is similar to accordion, but
    • accordion is showing options and items
    • module tag is show information and content
  • Add :active on the showing tag to increase accessibility

Navigation Tabs

  • What it solve
    • Navigate through a site to locate content and features
    • Clearly indicate user current location in the site
  • Note
    • It is Module Tabs with scope is the whole site

Navigation Bar

Provides persistent navigation assistance

Top Navigation Bar

Similar to Navigation Tabs, but it is a bar

  • Note
    • Navigation bar is in header
    • We can have a second level bar under the top level navigation bar
    • If there are second or third level bar, we can consider using left navigation bar
  • When to use
    • When the categories is kind of individual

Left Navigation Bar

Similar to Top Navigation Bar. It is better than Top Navigation Bar when:

  • There are more categories
  • The categories title is long
    • There more space to show the navigation bar
  • The category labels can be an expend button or hyperlink
    • Make the behaviour consistent

Progress Bar

The UI is similar to Breadcrumbs, but their purpose is different

  • What it solve
    • Tell users where they are in a multi-screen process
  • Note
    • The tag need to be an action
    • Don’t use this in some common or routine missions
    • Let user to estimate the length of flow

Select

Autocomplete

  • What it solve
    • Help user to enter input into a text box which could be ambiguous or hard to remember
    • Prevent typo
  • When to use
    • The suggestion can be pulled from a manageable set of data
    • The input item can be entered in multiple ways
    • Input speed and accuracy is important
    • The required input is not very long
  • Note
    • It gives lots of suggestion on interaction, but I don’t think we need to implement it

Calendar picker

Use calendar picker library. The library can handle this for us

Carousel

Provide a better object selection experience.

  • When to use
    • Image oriented menu
  • How it solve
    • Provide focus
    • User can slide
    • Order the items
  • Note
    • Need to show other options to indicate the carousel is slidable
    • Just use a carousel plugin

Rich Interaction

Invitation

Indicator there are extra feature on page element. It is useful when AJAX is not very common. I think it becomes common sense now.

Cursor Invitation

Change the icon of pointers

  • When to use
    • User can have interaction with some objects
    • Difficult to provide preview
    • Don’t need to provide preview
    • Want to simplify the operation

Drop Invitation

Indicate valid candidate drop sites.

  • Note
    • We can have more clear instrcution by using javascript

Hover Invitation

The description need to be short

Tool Tip Invitation

Show a short description when mouse hover on the page element

Transition

  • Let users understand the system is processing
  • Let users feel they need to wiat shorter
  • Make the UI more natural, because thing do not disappear in real world
  • User focus on changing element
  • Most transition should less then 0.5 second
  • When the change of element is not very important, we can use transition with 1 second duration

Animate Transition

Animating the in-between positions of a moving object.

  • When to use
    • Make the transition of hiding element more smooth
    • Show how an element change the position
    • Resizing element
  • Note
    • Don’t add animation which is not trigger by user.
      • It distraction the user

Bright transition

Singal that a previously dimmed object is now active or avaliable.

Collapse Transition

Hide an object which is no logner of primary importance.

Cross Fade Transition

Smoothly replace an old element with new element.

  • Fade out the old element and Fade in the new element
  • User can focus on the same area, but with content updated
  • I feel like this is outdated

Dim Transition

Indicate an object is secondary importance, not available or currently inactive. If it will not be available or active later, hiding it may be a better option.

Expand Transition

Indicate the previously collapsed object become active or available now. It can get more attention than Dim and Highlight transition.

Fade In Transition

Note

  • Duration; 0.25 to 0.5
  • If you need to show a secondary importance object, we can use fade in
  • Useful when you don’t want the transition catch too much attention

Fade Out Transition

It is used in self-healing Transition. The property is similar to Fade In Transition

Self Healing Transition

We can use this when some object is removed and leaving a hole in the list or form. It use slide or fade out to make resize more smooth.

Slide Transition

Note

  • Replace popup window
  • When the content is context dependent, use this to prevent context switch

Spotlight Transition

  • Have the highlight appear instantly when the change occurs and then fade out within one second or less
  • Useful to catch user’s attention

Drag and Drop

  • Useful for reorder layout without reload the page
  • There lots of detail, but we usually use library. So we can skip it
  • Revise this when we need
