SaaS - Software as a slide

今日係參加嘅Angelhack同我想像中嘅Hackathon有好大出入。比起Hackathon，大部份嘅參加者更加似係參加緊Startup比賽。

Hackathon v.s. Startup 比賽

我本來諗住Hackathon係大家有一D有趣嘅問題或諗法，之後一齊討論下可以點樣用technology去解決，之後會起一個prototype出黎做demo。但係唔少人都係business background。佢地D idea都好正路同普通嘅Business idea，參加都主要係揾人幫手implement個prototype。當我地 pitch 完個idea之後，人地第一時間問點賺錢同埋有咩business model。

Software as a slide

當個Topic係做一個business idea嘅prototype，以唔訓教嘅前題下都係得24個鐘，好明顯係唔夠時間。咁佢地個prototype同我地engineer心目中嘅prototype係好唔同。

我地Engineer嘅prototype係minimal value product，起碼真user係用到下先。但係個比賽嘅limitation下，做到嘅野連product都稱唔上。本來我諗住冇backend，frontend hardcode少少都可以算係prototype，但係係個比賽嘅脈絡之下，咁樣己經係太複雜。好多Team都係target做一個披住Software皮嘅Slides用黎做presentation，連frontend D 野都係fake同hardcode嘅，可以係一個webpage去邊有少少link同javascript去demo個flow，甚至連個flow都係播animation，或者用photoshop做出黎嘅。講到尾，code for presentation，如果可以冇code E part，D人一定會Remove左E一部份。

High tech，為Present帶黎吹水

個比賽得果24個鐘，好明顯做唔到D咩針對問題嘅research，咁一D高科技嘅野就可以係Demo任吹。例如個topic係做Machine learning on big data，咁個demo是但hardcode D data，之後用D template整靚個webpage就得。一個鐘搞掂，返屋企訓返教，第二日再準備present都仲得。點樣收集data，點用做machine learning真係知少少就可以是但吹，最多google下D API名同algo名交下貨。最近block chain比較熱門，但係present得果幾分鐘，解釋點解用block chain己經冇時間，更唔好講你個prototype可以點樣運用。

個人感受

有少少感覺自己去錯地方，但係同成班friend一齊整D side project係幾開心，以宏觀D嘅角度，係可以學到唔少野。

冇岩用嘅Skill

上文提到，做一個用software皮既slide係效益最大嘅。好明顯D咩software engineering嘅skill都唔啱洗，咩code design，咩reusability，咩algorithm都係唔岩用。我仲要係擅長做backend嘅，真係收得皮。

一齊brain storm低能idea

同班friend一齊brain storm低能idea係開心嘅，但係真係冇諗過會有街外人加入。一齊討論激發左D火花之後，個低能idea又好似work work地，成個experience係得意同開心嘅。

Programming以外嘅insight

當個idea俾一個完全唔同background嘅人，用完全唔同嘅角度去質疑自己嘅諗法，係可以刺激到好多思考，得到好多啟發。係成個weekend，我揾到自己嘅職業病同盲點，並identify到自己以一個full stack engineer嘅身份做product所缺乏嘅skill set。E D野都唔係睇書或者自己做project可以學到嘅野

至於實際上有咩learning同insight，就遲D得閒再寫一篇文去解釋。

比賽冠軍

比賽過後，同William仔細討論(主要都係我問，William答我)過後，HKU果條Team拎冠軍都係實至名歸。

Writio, an eassy quoting chrome extension

冠軍隊由HKU嘅學生組成。佢地嘅作品係一個chrome extension，透過highline可以save低網頁上嘅內容，之後可以係extension果個page drag入去google doc，咁就有一個靚靚仔仔嘅qoute係essay入面。

目標鮮明簡潔

個Product用一句就可以清楚解釋就証明左個idea係清淅明確，完全冇D咩複雜嘅logic係背後，唔洗解釋個product係做D咩，係2分鐘嘅present時間起左關鍵性嘅作用。

Product用法直接，冇learning curve

成個操作都係hightlight，right click同drag and drop，完全唔洗解釋點樣用，齋睇就得，大部份嘅時間可以用黎講product function同feature。成個presentation可以隻眼睇住demo，對耳聽住人地pitch，成個flow非常smooth。

Product完成度高

由於唔需要D咩高深技術，而且用途同操作都簡單，相信係24小時內係可以搞掂，而且係demo入面，個product靚仔，function到，基本上係個prototype係product，已經完成左。

學生身份有加分

比賽前邀請左上年冠軍share，佢話以學生身份係有著數，易D得到機會。而HKU team係present第一句就係”We are students from HKU”，係presentation一開波用左三秒已經攞到個buff。而且成個product係以學生身份design，令人印象深刻。雖然唔係關鍵勝負位，但係可以有效運用優勢，係評判心中留下好印象。

完全冇提到business

成個Demo係冇講過user同business有關嘅野，可能係同學生身份有關，但E條Team攞到冠軍，真係正面打左D以business為題材嘅Team塊面一野。
