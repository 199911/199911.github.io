Why null reference is a bad idea

  • Reason
    • It means “machine thinking”, over “Object Thinking”
      • If the code is business logic, then it makes sense
    • Needless complexity
    • Meaningless state
      • Not meaningless at some case, null means unknown in “Yes, No and unknown”
    • It’s not a Fail Fast approach
      • If we can throw error and don’t care the execution flow is break there
  • Solution
    • Throw error instead of return null
