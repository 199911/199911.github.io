今日睇左一堆文，冇咩心機每篇都仔細去諗，寫一份感想。但係又有好多零碎嘅 insight，因些放曬係度，等得閒果時再睇返。

为什么你应该（从现在开始就）写博客

文中提到，千析唔好係個blog到碎碎念，要放D organize左嘅knowledge上去。E一點幾衝擊我，因為我個blog D 野都幾零碎。但仔細咁諗，我主要係想有個地方記錄平時D insight，廢事自己唔記得。因此我暫時都會keep住E個style，當有足夠嘅insight嘅時候，再整理出一D 高質嘅文章，放係feature個分類入邊都未遲。

Comment

Aug 27, 2018:

寫一D零碎嘅文章去kick start係唔錯，但係2年後，自己睇返都好辛苦，好難organize返D知識。

  • 想Jot低insight，可以用返GTD system，更加適合
    • 可惜當時未接觸到GTD
  • 將文章organize得好D，對自己嘅學習同記憶都有幫助

锤子和钉子

講出programmer有簡單複雜化嘅盲點，特別係岩岩學左新野同有新insight嘅時候特別嚴重。同 second system effect有D似。講到尾最重要係認清objective，用最efficient嘅方法去解決問題。

Comment

Aug 27, 2018:

文章好中point咁講左新手嘅common mistake，仲有D好生動嘅故事

  • Over engineering
    • Second system effect 係一種造成over engineering嘅現象
    • 照跟教科書做法好易over engineering
      • 要針對解決現實問題去思考
  • 以現有嘅工具或技術去解決問題，而冇考慮到其他alternative solution
    • 重用現有工具或技術的確可以慳到時間，唔係唔岩。
    • 但係俾現有嘅野限制住，好易miss左其他oppertunity，好易miss左更好嘅solution
      • 變左食老本，好難進步

git-flow

提供一個git work flow 嘅abstraction，唔洗打一堆git command 先開到新branch。但係個design係俾每人fork一個repo嘅use case，到我地公司大家share 一個 repo個情況唔同，好似唔係幾岩用。
